Description

Regular expressions are very helpful when you have to find a match using some pattern. How many of them could be there? There is an infinite number of them, indeed we can't memorize all of them. Why don't we make a computer store all the information for us? This is the most efficient way, and we will have a preview of every input that the user has provided, and the result of comparing it to the string.

To accomplish that, we need to connect a database.  

Objectives

To begin with the project, we should really think of how all the data would be organized. In our case, user inputs truly matter. If you look at the pattern matching itself, you will realize that we have only two things that users should provide us: some pattern and some text to compare to. That's what we need to record! Furthermore, we are expecting the outcome of the comparison, so we can understand if the pattern is any good. And, we must keep these things connected to each other to make sure the result isn't from the different test. That's where we come up with the data model.
