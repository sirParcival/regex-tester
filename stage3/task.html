Description

Now our service takes input but nothing happens. Let's change it! This time, the user should get a response from the service, after filling up the form with the template and string and submitting it.

Objectives

We need to match the pattern and string provided by the user. Again, it's good not to do all the work manually. Besides, why try to match the pattern ourselves if we have a module that was created specifically for such a task?

Your task is to create a handler for the form using the re module. Firstly, you have to import this module and get the regex and string from the POST request in the view function. Then, the handler should try matching the submitted regex to the string and show the result to the user. At this stage, it should return an HttpResponse from django.http with True or False. Be careful! The matching method does not return a boolean by itself. At this stage, we use HttpResponse only to check the result.

Furthermore, right after testing a pattern, the service must save it to the database. Use a previously created Record model to do that. Remember, the result field is boolean, and the form method has to be POST.

Examples

For instance, the user entered ^[a-z0-9_-]{3,16}$ to match an alphanumeric string for their username with length from 3 to 16 symbols. As a string, the user enters thrawn_66. After submitting, the service should return True.

Another example: user wants to check the age. First input is [\d]{1,2}$ . It should match positive numbers with lengths from 1 to 2. When the user enters for the string field twenty, the result of a test must be False.
