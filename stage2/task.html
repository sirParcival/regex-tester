Description

Wonderful! Now, when we have where to store tests, what could we do next? Apparently, there should be some page with form to get inputs from users. With that simple solution, anyone will be able to try out their knowledge.

Objectives

Basically, you build a web page using HTML/CSS/JS. Pages that do not change information on them, are called static. For example, if you need to create a simple one-page portfolio, where you just want to place information about yourself, you only write text in your HTML document, and it is shown in your browser. But this method isn't perfect when you're creating a blog, for instance because it would be better not to create pages for each post. The solution for this is creating dynamic pages using templates. In our application, we will have a couple of pages that don't change their layout, but whose content changes constantly. You really don't want to write every page yourself, that's where templates become handy.

This time, the task is just to create a home page as an HTML template and connect Django view to it. The template should be with a simple layout:

  • At the top of the page, we will place a greeting message: <h2> Welcome to regex testing tool! </h2>.
  • Next, there should appear a main form with two text fields and a submit button. Don't forget to put csrf_token in your form! For your input tag, you have to use attribute name=... because you will access data from the form using these names. Please use 'regex' and 'text' for name attributes.
  • And at the bottom, add a link to the history page: <a href="/history/"> History </a>.

This main page needs to be available under GET request to localhost:8000. Create a view that renders a template for it. To complete that, use django.shortcuts.render function: render(request, template_name).

If you want to use a custom directory for your templates, add its path to the TEMPLATE['DIRS'] list in the settings.py module.

Don't worry, we'll implement handlers in the next stages.

If you start the application on your computer with the python manage.py runserver command, the main page should be available at the address localhost:8000.
